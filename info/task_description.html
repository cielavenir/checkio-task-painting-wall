You are given the number of meters to paint the wall and the list of the range to paint.
You must paint following the order described in the list (i.e. sorting etc will break this rule).
If the range to paint is overlapped, they must be counted only once.
You need to find the minimal number of painting to paint that range.

range to paintconditionpainted
[1,5]                           5
[11,15]                           10
[2,14]                           15
[21,25]                           20

Input: Number of meters to paint. And list of the range to paint the wall (inclusive).
Note that the range is very large (1<= range < 2*1018) in some test cases. Remembering all painted area is insufficient.

Output: Minimal number of operation. If you cannot paint enough range, return -1.

Example: 

checkio(5,[[1,5],[11,15],[2,14],[21,25]]) == 1 # The first operation will paint 5 meter long.
checkio(6,[[1,5],[11,15],[2,14],[21,25]]) == 2 # The second operation will paint 5 meter long. The sum is 10.
checkio(11,[[1,5],[11,15],[2,14],[21,25]]) == 3 # After the third operation, the range 1-15 will be painted.
checkio(16,[[1,5],[11,15],[2,14],[21,25]]) == 4 # Note the overlapped range must be counted only once.
checkio(21,[[1,5],[11,15],[2,14],[21,25]]) == -1 # There are no ways to paint for 21 meters from this list.

checkio(1000000011,[[1,1000000000],[11,1000000010]]) == -1 # One of the huge test cases.

Hint: To handle start-end list, using binary search is good idea. Since CheckiO Python2.7 doesn't allow bisect module, I recommend Python3.
# Author's solution has been locally tested using the same code on Python 2.7 and 3.3.

How it is used: Some kind of simulation.
